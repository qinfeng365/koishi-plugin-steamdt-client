## 提示和技巧

### 技巧1：快速检查服务
```
/health
```
如果返回健康状态，说明服务正常。

### 技巧2：监控队列
```
/queue
```
定期运行此命令，了解任务处理进度。

### 技巧3：查看最新任务
```
/task 0
```
通常最新完成的任务索引为0。

### 技巧4：使用优先级
- 紧急任务：优先级8-10
- 普通任务：优先级4-7
- 后台任务：优先级0-3

### 技巧5：合理设置滚动次数
- 快速测试：5-10次
- 标准爬取：15-20次
- 完整数据：30-50次
